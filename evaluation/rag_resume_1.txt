안녕하세요, 3년차 백엔드 개발자 김민수입니다. 대학교에서 컴퓨터공학을 전공했고, 졸업 후 스타트업에서 백엔드 개발자로 근무하며 실무 경험을 쌓아왔습니다. 사용자 중심의 안정적인 서비스를 만드는 것을 목표로 삼고 있으며, 새로운 기술을 배울 때는 공식 문서를 먼저 읽고 작은 토이 프로젝트로 직접 만들어보면서 익히는 편입니다. 막히는 부분은 커뮤니티나 블로그를 참고하며 스스로 해결하려고 노력합니다.

주요 기술 스택은 Python(FastAPI, Django), PostgreSQL, Redis, Docker입니다. 최근에는 AWS 환경에서 서비스를 운영하며 EC2, RDS, S3를 다뤄본 경험이 있고, CI/CD 파이프라인 구축을 위해 GitHub Actions도 사용해봤습니다. 코드 품질 관리를 위해 pytest 기반 테스트 코드 작성을 습관화하고 있고, 정적 타입 검사를 위해 mypy도 프로젝트에 도입해본 적이 있습니다. 최근에는 메시지 큐인 Kafka를 학습해서 사내 알림 시스템 일부에 적용해보기도 했습니다.

가장 주도적으로 진행한 프로젝트는 사내 B2B 업무관리 시스템 개발이었습니다. 동시에 여러 사용자가 같은 예약 리소스를 수정할 때 데이터 정합성이 깨지는 문제가 있었는데, PostgreSQL의 SELECT FOR UPDATE로 행 단위 잠금을 적용해 해결했습니다. 이후 트랜잭션이 길어지며 발생하는 데드락 문제도 잠금 순서를 통일해서 개선했습니다. 이 프로젝트는 4명 팀으로 진행했고, 저는 예약 도메인과 결제 연동 부분을 담당했습니다.

또한 자주 조회되지만 잘 바뀌지 않는 데이터를 Redis에 캐싱해서 평균 응답 시간을 400ms에서 80ms로 줄인 경험이 있습니다. 캐시 무효화 전략은 TTL 기반으로 설계해서 데이터 신선도 문제도 함께 관리했습니다. 이 개선으로 서버 인스턴스 비용도 약 20% 절감할 수 있었습니다. 트래픽이 몰리는 시간대에 DB 커넥션 풀이 고갈되는 문제도 겪었는데, 커넥션 풀 크기를 조정하고 오래 걸리는 쿼리를 별도 워커로 분리해서 해결했습니다.

귀사에 지원한 이유는 실시간 데이터 처리를 중요하게 다루는 서비스 구조에 관심이 생겼기 때문입니다. 제가 이전 프로젝트에서 쌓은 캐싱과 동시성 제어 경험이 귀사의 서비스 확장에 기여할 수 있을 것이라 생각합니다. 입사 후 3년 안에는 대용량 트래픽 환경에서도 안정적으로 동작하는 백엔드 아키텍처를 설계하는 시니어 개발자로 성장해서, 팀의 기술적 의사결정에 기여하고 싶습니다.

팀 프로젝트에서는 4명 팀의 백엔드 파트 리더를 맡아 일정 관리와 코드 리뷰를 주도했고, 매주 짧은 스크럼으로 진행 상황을 공유했습니다. 팀원과 기술적으로 의견이 갈렸을 때는 먼저 상대방의 근거를 끝까지 듣고, 각자의 장단점을 정리해서 비교한 뒤 팀 목표에 더 부합하는 방향으로 합의점을 찾으려고 노력합니다. 코드 리뷰에서 "가독성보다 방어적 코드에 너무 치우쳐 있다"는 피드백을 받은 뒤로는, 예외 처리 범위를 필요한 만큼만 좁히려고 신경 쓰고 있습니다.

저의 강점은 문제를 끝까지 파고드는 끈기이고, 약점은 완벽을 추구하다 일정을 빠듯하게 잡는 편이라 최근에는 일정에 여유 버퍼를 두려고 노력하고 있습니다. 스트레스를 받을 때는 가벼운 러닝을 하면서 머리를 비우거나, 문제를 작은 단위로 쪼개서 하나씩 해결해나가는 방식으로 풀어갑니다.
